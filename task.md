GitHub Pages で俳優のポートフォリオサイトを作るのは、素晴らしいアイデアですね！
データの更新（出演作の追加など）を Markdown や YAML で管理する構成にすれば、コードをいじらずにサイトの内容をサクッと更新できる、運用しやすいサイトになります。

2026 年現在のトレンドとメンテナンス性を考慮した、最適な実装プランを提案します。

---

## 1. 推奨される技術スタック

GitHub Pages との親和性が高く、Markdown/YAML の扱いが最も得意な**Astro**をベースにするのがおすすめです。

- **フレームワーク:** [Astro](https://astro.build/) (高速で、Markdown/YAML のデータ取得が標準機能として強力)
- **スタイリング:** Tailwind CSS (直感的にデザインでき、レスポンシブ対応も容易)
- **デプロイ:** GitHub Actions (リポジトリに push するだけで自動公開)

---

## 2. サイト構成案

俳優のサイトとして必要な要素を整理しました。

| ページ名    | 内容                                             | データ形式      |
| ----------- | ------------------------------------------------ | --------------- |
| **Home**    | インパクトのあるメインビジュアル、最新のお知らせ | YAML            |
| **Profile** | 基本プロフィール、特技、サイズ、所属事務所       | Markdown        |
| **Works**   | 出演実績（映画、ドラマ、舞台など）の一覧         | YAML            |
| **Gallery** | 宣材写真、舞台写真                               | YAML (画像パス) |
| **Contact** | お問い合わせ先（SNS リンク、事務所フォームなど） | Markdown        |

---

## 3. 実装ステップ

### STEP 1: リポジトリの準備

GitHub で「Public」リポジトリを作成します。

### STEP 2: データ構造の作成 (ここが肝心！)

`/src/content/` フォルダ内にデータを配置します。

- **経歴データ (`works.yml`)**:

```yaml
- title: "映画「青空の向こう側」"
  year: 2025
  role: "主演：佐藤 翼 役"
  category: "Movie"
- title: "舞台「夏の夜の夢」"
  year: 2024
  role: "パック 役"
  category: "Stage"
```

- **プロフィール (`profile.md`)**:

```markdown
---
name: "山田 太郎"
skills: ["アクション", "乗馬", "英会話"]
---

俳優として、映像を中心に活動中。
1995 年生まれ、東京都出身。
```

### STEP 3: デザインの実装

Astro の `getCollection` 関数を使って、YAML や Markdown を自動で読み込み、HTML に変換するコンポーネントを作成します。これにより、**「データを増やせば自動でページが増える」** 仕組みが完成します。

### STEP 4: 自動デプロイの設定

GitHub Actions を使って、`main` ブランチに push した瞬間にビルドと公開が走るように設定します（`.github/workflows/deploy.yml`）。

---

## 4. 俳優サイトならではのポイント

> [!TIP] > **画像サイズに注意！**
> 俳優ポートフォリオは写真が命ですが、高画質な写真をそのまま上げるとサイトが重くなります。Astro の `assets` 機能を使えば、自動で WebP などの軽量形式に最適化してくれるので、ぜひ活用しましょう。

- **OGP 設定:** SNS でシェアされた時に、あなたの顔写真と名前がきれいに表示されるように設定します。
- **PDF 書き出し:** Markdown データを流用して、そのまま「紙のプロフィール（香盤）」風の PDF を生成する仕組みを作ると、オーディション時に重宝します。
